# Внесок у розробку Nova Poshta SDK

Дякуємо за інтерес до покращення цього SDK! Ми вітаємо будь-які внески, від виправлення опечаток до додавання нових функцій. Ось кілька рекомендацій, які допоможуть зробити процес внеску максимально ефективним.

## Підготовка середовища розробки

1. Форкніть репозиторій на GitHub.
2. Клонуйте форкнутий репозиторій на ваш комп'ютер:
   ```bash
   git clone https://github.com/[ваше-ім'я]/nova-poshta-sdk.git
   cd nova-poshta-sdk
   ```
3. Встановіть залежності:
   ```bash
   composer install
   ```
4. Створіть нову гілку для вашої функції або виправлення:
   ```bash
   git checkout -b feature/назва-функції
   ```

## Стиль коду

Проект використовує PSR-12 стандарт кодування. Будь ласка, дотримуйтесь цих стандартів при написанні коду.

Для перевірки стилю коду використовуйте:
```bash
composer cs-check
```

Для автоматичного виправлення стилю:
```bash
composer cs-fix
```

## Тестування

Для будь-яких змін у коді необхідно писати тести. Проект використовує PHPUnit для тестування.

Запуск тестів:
```bash
composer test
```

Перевірка типів за допомогою PHPStan:
```bash
composer phpstan
```

## Додавання нових функцій

1. Переконайтеся, що нова функція відповідає офіційному API Нової Пошти.
2. Додайте відповідні методи у відповідні класи модулів API.
3. Напишіть тести для нових методів.
4. Оновіть документацію, щоб відобразити нові можливості.

## Структура та організація коду

- Код SDK знаходиться в директорії `app/`
- Тести знаходяться в директорії `tests/`
- Кожен модуль API має свій власний клас в директорії `app/Api/`

## Процес відправки змін

1. Закомітьте ваші зміни:
   ```bash
   git add .
   git commit -m "Опис ваших змін"
   ```
2. Синхронізуйте вашу гілку з останньою версією оригінального репозиторію:
   ```bash
   git remote add upstream https://github.com/aunhurian/nova-poshta-sdk.git
   git fetch upstream
   git rebase upstream/main
   ```
3. Відправте зміни у ваш форк:
   ```bash
   git push origin feature/назва-функції
   ```
4. Створіть Pull Request на GitHub.

## Правила для Pull Request

1. Назва PR повинна коротко описувати зміни.
2. Опишіть детально, що саме було змінено або додано.
3. Посилайтеся на відповідні issues, якщо вони є.
4. Переконайтеся, що всі тести проходять успішно.
5. Переконайтеся, що документація оновлена.

## Повідомлення про помилки

Якщо ви знайшли помилку, будь ласка, створіть issue на GitHub з детальним описом:

1. Що сталося і що ви очікували
2. Кроки для відтворення помилки
3. Версія PHP та інформація про середовище
4. Код, який викликав помилку (якщо можливо)

## Функції API Нової Пошти

При додаванні нових методів для API Нової Пошти:

1. Перевірте, що метод існує в [офіційній документації API](https://developers.novaposhta.ua/documentation)
2. Дотримуйтесь конвенцій найменування, використаних в SDK
3. Додайте параметри методу з відповідними типами
4. Додайте PHPDoc з описом параметрів та повертаємого значення
